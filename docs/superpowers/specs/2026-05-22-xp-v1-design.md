# xP v1 — Design Spec

- **Date:** 2026-05-22
- **Status:** Approved for planning
- **Scope:** Second half of the Analytics layer — the Expected Points (xP) v1 model. Consumes FDR v1 + Understat per-player rates + minutes. (Decision Layer — captain/transfer/chip — comes after, and can fan out to parallel agents once this lands.)
- **Slice goal:** the `xp` table holds an inspectable per-player, per-upcoming-GW expected-points value (`model_version='v1'`), computed deterministically from data already in the DB.

Builds on the merged FDR v1 slice and the Understat data layer.

---

## 1. Context

`decision-engine.md` defines xP v1's structure. Translating it to concrete code surfaced two issues, both resolved here and recorded in `decision-engine.md` (B4 — decision logic is sacred, documented before code):

1. **`xMinutes`-as-addend is a doc typo.** The doc's sum literally adds raw `xMinutes` to a points total, which would swamp everything (a player's xP would be ~75). The minutes contribution to *points* is `appearance_points` (1 for playing, 2 for 60+). `xMinutes` only **scales** xGoals/xAssists (`× xMin/90`) and **gates** the clean sheet (`× p60`). Corrected here.
2. **Clean sheet can't use Poisson(opponent xG/90).** Team xG is unavailable (the reason FDR is FPL-strength v1). Clean-sheet probability is derived from `fdr_defense` instead — consistent with FDR v1.

All xP inputs are already in the DB (Analytics reads only the DB, never the network — B2): `understat_players` (`xg_per_90`, `xa_per_90`, `minutes`, `games`, `fpl_player_id`), `fdr` (`fdr_attack`, `fdr_defense` per team/GW), `players` (`position`, `status`, `team_id`).

## 2. Decisions locked

| Decision | Choice | Rationale |
|---|---|---|
| Minutes contribution | `appearance_points` (0–2), NOT raw `xMinutes` | Doc typo fix; raw minutes would dominate xP. |
| Clean sheet | `cs_prob(fdr_defense)` × `p60` | Team xG unavailable; consistent with FDR v1. |
| Attack fixture effect | `fdr_attack_mult[1..5]` = 1.20/1.10/1.00/0.90/0.80 | ±20% around neutral; versioned, tunable. |
| Clean-sheet rates | `cs_prob[1..5]` = 0.55/0.45/0.35/0.22/0.12 | FPL-realistic; versioned, tunable. |
| xMinutes source | `understat.minutes / understat.games`, status-adjusted, capped 90 | We have season totals, not per-GW; a fine v1 proxy. |
| Coverage | matched players only (need xg/xa per 90 + minutes) | ~520 players incl. the whole realistic pool; unmatched degrade safely. |
| Versioning | store `model_version='v1'` + components | B5 (version xP) + B4 (decisions inspectable). |

## 4. xP v1 formula (precise)

Constants:
```
GOAL_PTS = {GKP:6, DEF:6, MID:5, FWD:4}
CS_PTS   = {GKP:4, DEF:4, MID:1, FWD:0}
STATUS_MULT = {a:1.0, d:0.5, i:0.0, s:0.0, u:0.0}   # unknown -> 1.0
FDR_ATTACK_MULT = {1:1.20, 2:1.10, 3:1.00, 4:0.90, 5:0.80}
CS_PROB         = {1:0.55, 2:0.45, 3:0.35, 4:0.22, 5:0.12}
```
Per player, per GW (given the player's team's `fdr_attack`/`fdr_defense` for that GW):
```
xMin       = min(minutes/games, 90) * STATUS_MULT[status]      # 0 if games==0
p_appear   = clamp(xMin/20, 0, 1)
p60        = clamp((xMin-30)/30, 0, 1)
appearance = p_appear + p60                                    # 0..2
xGoals     = xg_per_90 * (xMin/90) * FDR_ATTACK_MULT[fdr_attack]
xAssists   = xa_per_90 * (xMin/90) * FDR_ATTACK_MULT[fdr_attack]
xCleanSheet= CS_PROB[fdr_defense] * p60
xP         = appearance + xGoals*GOAL_PTS[pos] + xAssists*3 + xCleanSheet*CS_PTS[pos]
```
Stored (rounded): `xminutes=xMin`, `xgoals=xGoals`, `xassists=xAssists`, `xcs=xCleanSheet`, `xp=xP`.

**Horizon:** next 6 GWs from the first unfinished GW (`MIN(id) FROM gameweeks WHERE finished=0`), same as FDR. A player gets an xP row for each horizon GW where their team has an FDR row.

## 7. Components

### `src/analytics/xp.py`
- Module constants (§4).
- `compute_player_xp(position, status, xg90, xa90, minutes, games, fdr_attack, fdr_defense) -> dict` — **pure**; returns `{xminutes, xgoals, xassists, xcs, xp}` (rounded). `games==0` → `xMin=0` (all-zero result). Unknown status → mult 1.0.
- `compute_and_store(conn, horizon=6) -> int` — finds the next unfinished GW; loads matched players (join `players`⋈`understat_players` on `fpl_player_id`) and the `fdr` rows for the horizon; for each (player, gw) where the player's team has an FDR row, computes xP and persists via `repository.upsert_xp` with `model_version='v1'` + `computed_at`. Returns row count.

### `repository.upsert_xp(conn, rows)`
Upsert into `xp (player_id, gw, model_version, xp, xminutes, xgoals, xassists, xcs, computed_at)`, `ON CONFLICT(player_id, gw, model_version)`.

## 8. Testing (B11)
- `test_compute_player_xp_striker_easier_fixture_scores_more`: same striker inputs, `fdr_attack=1` yields higher `xp` than `fdr_attack=5`.
- `test_compute_player_xp_gk_earns_clean_sheet`: a GKP with `xg90=xa90=0`, full minutes, `fdr_defense=1` gets `xp > 0` driven by `xcs` (and appearance).
- `test_compute_player_xp_position_weighting`: a DEF goal worth 6 vs a FWD goal worth 4 — identical xg inputs, DEF `xgoals*6` > FWD `xgoals*4` reflected in xp.
- `test_compute_player_xp_status_injured_zero`: `status='i'` → `xMin=0` → `xp == appearance(0) + 0 == 0`.
- `test_compute_player_xp_zero_games_safe`: `games=0` → no division error, xp 0.
- `test_compute_and_store_persists` (integration, in-memory DB): seed `teams`/`players`/`understat_players`/`fdr`/`gameweeks` (frozen bootstrap + a crafted unfinished GW with fdr rows), run `compute_and_store`, assert `xp` rows exist with `model_version='v1'`, components populated, idempotent on re-run.

## 9. Definition of done
1. `pytest` green incl. the xP tests.
2. `compute_and_store` populates the `xp` table for the horizon against the live DB, values sane (premium attackers on easy fixtures rank highest; GKs/defenders get clean-sheet-driven xP), idempotent.
3. `decision-engine.md` updated with the concrete xP v1 section + changelog (B4).

## 10. Notes / caveats
- v1 uses **season-average** rates, not rolling form — acceptable for v1; rolling needs per-GW data (deferred with FDR v2).
- Coverage is matched-Understat players only; unmatched fringe players get no xP (safe degradation).
- DGW/BGW not modeled (one fixture per team/GW assumed).
